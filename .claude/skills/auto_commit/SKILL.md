# Skill: Git Auto Commit & Push
## 描述
自动分析当前 Git 变更，生成规范 Conventional Commit 提交信息，展示给用户确认，用户同意后自动执行 git add、git commit、git push 完整流程。

## 触发指令
- 提交代码
- 自动生成 commit 并提交推送
- git 一键提交
- /auto_commit

## 执行规则

### 第一步：检查 Git 仓库状态
1. 验证当前目录是 Git 仓库：`git rev-parse --is-inside-work-tree`
2. 获取变更列表：
   ```
   git status --short
   git diff --stat
   ```
3. 判断是否有可提交内容，如无则提示并退出

### 第二步：排除无需提交的文件
检查变更文件中是否存在以下类型，如存在则从提交中排除：
- 编译产物：dist、build、target、out
- IDE 配置：.idea、.vscode
- 临时文件：*.tmp、*.bak、*.log
- 依赖缓存：node_modules、.npm、.yarn
- 环境变量：.env、*.local.json

### 第三步：更新本地分支（可选）
如果当前分支有远程追踪分支，先执行：
```
git pull --rebase
```
如无远程分支则跳过此步骤。

### 第四步：用户输入需求编号或者缺陷编号
1. 提示用户输入本次提交的需求编号或者缺陷编号（如 R2026031605820、B20241225002148）
2. 用户输入编号后，**不要直接继续**，等待用户明确确认
3. 重新展示变更文件列表，并在末尾明确询问：
   ```
   🤖 以上是本次变更的文件，请输入需求/缺陷编号（如 R2026031605820）：
   ```
4. 用户输入编号后，对变更信息进行**特殊符号检查**（见下方检查规则）
5. 特殊符号检查通过后，生成提交信息并展示，然后**换行重新询问确认**
6. 只有用户明确回答 y/是 后才执行第六步

### 特殊符号检查规则
用户输入的编号或生成的提交信息中**禁止包含**以下 Git 特殊符号：

| 符号 | 说明 |
|------|------|
| `#` | Git 中 `#` 开头会被忽略，消息不会提交 |
| `\` | 转义字符，可能导致解析错误 |
| `"` | Windows commit 信息引号，与外层冲突 |
| `'` | Unix 单引号可能在某些 shell 中有问题 |
| NUL (`\0`) | 空字符会截断消息 |

**检查流程**：
1. 如果用户输入包含上述符号，提示「输入包含 Git 特殊符号 {符号列表}，进行自动转换后继续」并自动替换（见下表）
2. **自动生成的 commit 信息也需要进行特殊符号审查**
3. 如果生成的信息模板包含上述符号，按以下规则**自动替换**：
   | 原符号 | 替换为 |
   |--------|--------|
   | `#` | 删除（Git 中 `#` 开头会被忽略） |
   | `\` | `/` |
   | `"` | `'` |
   | `'` | `"` |
   | `\0` | 删除 |
4. 检查通过后继续执行

### 第五步：生成并确认提交信息
用户输入编号后，生成提交信息并**完整展示**，然后明确询问：

```
---
生成的提交信息：
chore(R2026031605820): 清理旧的pipeline日志和决策文档
🤖 AI Generated

---
📝 是否使用该提交信息执行 add → commit → push？(y/是 或 n/否)
```

- 用户确认后（y/是），继续执行第六步
- 用户拒绝后（n/否），显示「已取消提交」并终止流程

### 第六步：执行提交
用户确认后执行以下命令：

1. **git add**：添加排除后的变更文件
2. **git commit**：
   - Windows 使用双引号包裹信息
   - Unix/Linux 使用单引号包裹信息
3. **git push**：
   - 检查是否存在远程分支
   - 如无远程分支，使用 `git push -u origin HEAD` 设置 upstream
   - 否则直接 `git push`

### 异常处理

| 场景 | 处理方式 |
|------|----------|
| 非 Git 仓库 | 提示「当前目录不是 Git 仓库」并退出 |
| 无任何变更 | 提示「没有需要提交的变更」并退出 |
| 未输入需求或者缺陷编号 | 提示「请输入需求或者缺陷编号」并返回重新输入 |
| git pull 冲突 | 提示存在冲突，需手动解决后重试 |
| commit 失败 | 打印错误信息，终止流程 |
| push 失败 | 打印错误信息，终止流程 |
| 用户拒绝 | 提示「已取消提交」，正常退出 |
