# Pipeline Eval

跨运行评估。用户主动调用，不修改任何文件，只产出诊断报告。

## 输入

$ARGUMENTS = 可选，默认评估最近 5 次运行，可指定如 "3" 评估最近 3 次

## 4 目标评估框架

每个目标由多个检查项组成，加权计分。**必须有执行证据才给分。**

### 目标 1: 编排正确性

| 检查项 | 权重 | 评分标准 |
|--------|------|---------|
| 阶段顺序正确 | 20% | 顺序与 CLAUDE.md 一致 |
| 无跳步/合并 | 15% | SR-0 全部合规 |
| 失败正确处理 | 15% | review 不通过后有修复重审循环 |
| State 正确更新 | 10% | SR-6 全部合规 |
| 日志结构完整 | 10% | 每个阶段有五段 + SR + QR + 观察 |

### 目标 2: 子 agent 质量

| 检查项 | 权重 | 评分标准 |
|--------|------|---------|
| 知识加载完整 | 15% | SR-1/SR-2 全部合规 |
| Act 步骤遵循 | 15% | SR-4 全部合规 |
| Verify 有证据 | 10% | SR-5 全部合规 |
| 无越界 | 10% | SR-7 全部合规 |

### 目标 3: 知识传递有效性

| 检查项 | 权重 | 评分标准 |
|--------|------|---------|
| Handoff 完成 | 20% | SR-3 全部合规（非首阶段） |
| References 可达 | 15% | 全部完整路径 |
| 知识回流触发 | 15% | SR-8 全部合规 |
| Review 独立性 | 10% | review 只读 state+产出+review.md |

### 目标 4: 产出可交付性

| 检查项 | 权重 | 评分标准 |
|--------|------|---------|
| 测试通过 | 20% | 测试零错误（非代码交付标记 N/A） |
| PR 质量 | 20% | 变更摘要+模块列表+知识变更 |
| Issue 关联 | 10% | Closes #N 正确 |
| 无无关文件 | 10% | git diff 无范围外文件 |
| 知识文件同步 | 10% | deliver 阶段确认知识文件变更 |

### 评分规则

- 每个检查项：满分/半分/零分
- 每个目标 = 加权求和
- 总评 = 4 个目标的平均值

## 执行步骤

### 1. 读取日志文件

```
.glob(.claude/pipeline-log-*.md)
```

### 2. 分析每个 Issue 的日志

- 验证阶段顺序：understand → design → plan → implement → deliver
- 验证 SR-0 ~ SR-8 合规率
- 验证 QR-0 ~ QR-3 质量
- 验证日志结构完整性

### 3. 产出诊断报告

按以下格式输出：

```
## Pipeline Eval 报告

### 总体评分
| 目标 | 评分 |
|------|------|
| 1. 编排正确性 | X/10 |
| 2. 子 agent 质量 | X/10 |
| 3. 知识传递有效性 | X/10 |
| 4. 产出可交付性 | X/10 |
| 总评 | X/10 |

### SR 合规率
| SR 项 | 合规率 |
|-------|--------|
| SR-0 | X/N |
| SR-1 | X/N |
| ... | ... |

### QR 合规率
| QR 项 | 合规率 |
|-------|--------|
| QR-0 | X/N |
| ... | ... |

### 系统性问题
- P1: {问题}
- P2: {问题}

### 改进建议
1. ...
```

## 约束

- 不自动修改任何文件
- 只分析和建议
- 报告可供后续引用
