# ADR: 反馈机制方案

## 背景

Issue #7 村委公示栏要求支持村民反馈（提交反馈意见）和工作人员回复。需确定反馈的处理流程是否需要审核机制。

## 备选方案

### 方案 A: 直接发布（无需审核）

- 优势: 流程简单即时，村民反馈立即可见；减少工作人员审核工作量；村民参与感更强
- 劣势: 可能出现不当内容（广告、人身攻击等）；需要后期内容监控手段

### 方案 B: 审核后发布

- 优势: 可过滤不当内容；保障公示评论区质量；符合政务系统内容合规要求
- 劣势: 增加审核环节，反馈不能立即可见；增加工作人员工作量；流程复杂化

## 决策

选择方案 A（直接发布），因为 MVP 阶段优先保证用户体验和功能完整性；村级公示栏为内部系统，用户群体为村民和工作人员，信任度较高；Boundary 明确"公示自动过期提醒"范围之外，未提及审核需求；后续如需审核可扩展，不影响核心功能实现。

## 否决项

- 方案 B: MVP 阶段增加审核流程会延长反馈响应时间，降低村民参与积极性；当前业务场景下审核非必需

## 影响

- notice_feedback 表无需 status 字段（默认为已发布）
- 反馈提交后直接入库，前端可立即查看
- 工作人员可对反馈进行回复，形成完整互动
- 后续扩展可增加审核状态字段
