# ISSUE-MD模板管理（优化版）
## 概述
实现Markdown模板的定义、管理、应用功能，支持用户自定义MD模板，在创建MD文件时快速引用模板内容生成新文件，集成主界面侧边导航，无权限配置，提升MD文件创建效率。

## 功能需求
### F-TEMPLATE-001 MD模板基础管理
| 需求项 | 内容 |
|--------|------|
| 需求名称 | MD模板基础管理 |
| 需求描述 | 支持MD模板的新增、编辑、删除、查询、启用/禁用 |
| 模板属性 | 模板名称、模板标识、模板内容、创建时间、状态 |
| 模板校验 | 模板名称唯一、模板内容非空、模板标识规范 |
| 状态管理 | 启用/禁用模板，禁用模板不可用于文件创建 |

### F-TEMPLATE-002 MD模板应用
| 需求项 | 内容 |
|--------|------|
| 需求名称 | MD模板应用 |
| 创建关联 | MD文件创建页面提供模板选择下拉框 |
| 模板预览 | 选择模板时实时预览模板内容 |
| 内容填充 | 选中模板后自动将模板内容填充到编辑器 |
| 模板覆盖 | 支持手动修改填充后的内容，不影响原模板 |

### F-TEMPLATE-003 模板高级功能
| 需求项 | 内容 |
|--------|------|
| 需求名称 | 模板高级功能 |
| 模板分类 | 支持按业务场景分类管理模板（会议纪要、需求文档、周报） |
| 模板搜索 | 支持按模板名称、标识、分类模糊搜索模板 |
| 导入导出 | 支持模板JSON格式导入/导出 |
| 默认模板 | 支持设置系统默认模板，新建MD自动填充 |

### 数据库设计
```sql
-- MD模板主表
CREATE TABLE t_md_template (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '模板ID',
    template_name VARCHAR(100) NOT NULL COMMENT '模板名称',
    template_key VARCHAR(50) NOT NULL UNIQUE COMMENT '模板标识',
    category VARCHAR(50) DEFAULT 'default' COMMENT '模板分类',
    content TEXT NOT NULL COMMENT '模板内容',
    is_default TINYINT DEFAULT 0 COMMENT '是否默认模板 0-否 1-是',
    status TINYINT DEFAULT 1 COMMENT '状态 0-禁用 1-启用',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    del_flag TINYINT DEFAULT 0 COMMENT '删除标识 0-未删除 1-已删除'
) COMMENT 'MD模板表';

-- 模板分类字典表
CREATE TABLE t_template_category (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    category_name VARCHAR(50) NOT NULL,
    sort INT DEFAULT 0,
    status TINYINT DEFAULT 1
) COMMENT '模板分类表';
