# ISSUE-8-新闻动态

## 概述

发布村级新闻、政策解读、活动报道、先进事迹、公示通知、视频展博等内容，支持图文编辑、分类标签设置（如"政策解读""活动报道"）前端展示通过单页面按类型进行分栏展示，头部可以展示图片轮播，每个分栏默认展示5条，展现形式为图片缩略图+标题；内容按发布时间排序，支持按分类、发布时间查询；村民可查看所有新闻动态，了解村级最新情况；村级工作人员可对新闻进行编辑、删除、归档。

## 功能需求

### F-NEW-001 发布新闻

| 需求项 | 内容 |
|--------|------|
| 需求名称 | 发布新闻 |
| 需求描述 | 发布新闻内容，支持图文编辑和分类标签 |
| 输入 | 标题、内容（支持图片）、分类标签 |
| 校验 | 标题非空，内容非空，分类有效 |
| 输出 | 新闻发布成功 |
| 异常处理 | 标题重复提示"已存在同名新闻" |

### F-NEW-002 编辑新闻

| 需求项 | 内容 |
|--------|------|
| 需求名称 | 编辑新闻 |
| 需求描述 | 修改新闻内容 |
| 输入 | 新闻ID、要修改的字段及新值 |
| 校验 | 新闻存在且未归档 |
| 输出 | 编辑成功提示，信息更新 |

### F-NEW-003 删除新闻

| 需求项 | 内容 |
|--------|------|
| 需求名称 | 删除新闻 |
| 需求描述 | 软删除新闻内容 |
| 输入 | 删除确认、新闻ID |
| 校验 | 新闻存在且未归档 |
| 输出 | 确认弹窗，成功后从列表移除 |

### F-NEW-004 新闻归档

| 需求项 | 内容 |
|--------|------|
| 需求名称 | 新闻归档 |
| 需求描述 | 将新闻标记为归档状态 |
| 输入 | 新闻ID |
| 校验 | 新闻存在 |
| 输出 | 归档成功 |
| 日志记录 | 归档人、操作时间 |

### F-NEW-005 新闻查询

| 需求项 | 内容 |
|--------|------|
| 需求名称 | 新闻查询 |
| 需求描述 | 按分类、发布时间查询新闻 |
| 输入 | 查询条件（分类/时间范围） |
| 校验 | - |
| 输出 | 符合条件的新闻列表 |
| 排序 | 按发布时间倒序 |

### F-NEW-006 分类管理

| 需求项 | 内容 |
|--------|------|
| 需求名称 | 分类管理 |
| 需求描述 | 设置新闻分类标签 |
| 输入 | 分类名称 |
| 校验 | 分类名称非空 |
| 输出 | 分类创建成功 |
| 预置分类 | 政策解读/活动报道/先进事迹/通知公告 |

### 数据结构

**新闻表 (news)**

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO | 新闻ID |
| title | VARCHAR(100) | NOT NULL | 标题 |
| content | TEXT | | 内容 |
| category | VARCHAR(20) | | 分类标签 |
| publisher | VARCHAR(50) | | 发布人 |
| is_archived | TINYINT | DEFAULT 0 | 是否归档 |
| publish_time | DATETIME | | 发布时间 |
| created_at | DATETIME | | 创建时间 |
| updated_at | DATETIME | | 更新时间 |

### 接口设计

| 接口 | 方法 | 路径 | 功能 |
|------|------|------|------|
| 发布新闻 | POST | /api/news/create | 发布新闻 |
| 更新新闻 | PUT | /api/news/update | 更新新闻 |
| 删除新闻 | DELETE | /api/news/delete/{id} | 删除新闻 |
| 获取新闻 | GET | /api/news/get/{id} | 获取新闻详情 |
| 新闻列表 | POST | /api/news/list | 新闻列表 |
| 新闻归档 | PUT | /api/news/archive/{id} | 新闻归档 |
| 分类列表 | GET | /api/news/category/list | 分类列表 |
| 创建分类 | POST | /api/news/category/create | 创建分类 |

### 业务规则

1. 分类标签：政策解读/活动报道/先进事迹/通知公告（预置，可扩展）
2. 归档后不可编辑
3. 内容支持富文本（图片插入）
4. 标题全局唯一性校验
5. 发布时间自动记录
