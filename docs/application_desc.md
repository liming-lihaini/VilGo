# 开发技术架构

# 一、文档概述

明确软件开发的技术架构、前后端约束、数据组件规范及工程初始化目录要求，为开发团队提供统一的技术标准和实施依据，确保开发过程规范、高效，贴合单机版、本地存储、无数据加密及备份的核心需求，适配JDK 17环境。

# 二、整体技术架构

本软件采用“前后端一体化（单工程）\+ 本地数据库”架构，将前端、后端服务整合在一个工程中，适配单机部署模式，无外网依赖，所有数据存储于本地，简化部署流程、降低运行门槛，贴合村级工作人员操作场景。整体架构分为前端模块、后端模块、数据层三大模块，各模块在同一工程内封装、联动运行，无需单独部署前端服务。

核心技术栈：JDK 17、（后端开发）、SQLite 3（本地数据库）、Vue 3（前端）Vite、Maven（项目构建），无第三方加密、备份组件，聚焦核心业务功能实现，前后端一体化部署，无需额外配置前端服务。

# 三、前后端约束（一体化工程）

## 3\.1 后端约束

- 开发环境：必须使用JDK 17，兼容Windows 10/11操作系统，禁止使用低于JDK 17的版本，避免版本兼容问题。

- 开发框架：采用Spring Boot 2\.7\.x（稳定版），简化配置，提升开发效率，不引入Spring Cloud等分布式相关组件（贴合单机版需求）；集成Spring Boot Web模块，负责后端接口开发及前端资源托管。

- 接口规范：采用RESTful风格接口，统一接口命名规范（如GET /api/v1/resident/query 用于查询村民信息），接口返回格式统一为JSON，包含code（状态码）、message（提示信息）、data（返回数据）三个核心字段。

- 编码规范：遵循Java编码规范（阿里巴巴Java开发手册），类名、方法名、变量名命名规范统一，注释完整（类注释、方法注释、关键代码注释），避免冗余代码。

- 依赖约束：仅引入核心依赖（Spring Boot Starter Web、SQLite驱动、Lombok、前端资源打包相关依赖等），禁止引入无关依赖，避免增加系统体积和运行负担；不引入任何数据加密、备份相关依赖（如加密工具包、备份组件）。

- 运行约束：后端服务启动后，自动关联本地SQLite数据库，同时托管前端静态资源，无需手动配置数据库连接和前端服务；服务启动失败时，输出明确的错误提示（如数据库连接失败、端口占用等），便于排查问题。

## 3\.2 前端约束（一体化工程）

- 开发框架：采用Vue 3 \+ Vite \+ Element Plus，贴合中老年用户操作习惯，界面简洁、按钮清晰，减少复杂交互，适配PC端和移动端（本地联动，无需外网同步）；前端代码整合在后端工程中，通过Maven打包统一部署。

- 样式规范：统一UI样式（颜色、字体、按钮大小、间距），避免个性化样式冗余；适配不同屏幕分辨率，确保在Windows 10/11常用分辨率下正常显示。

- 交互约束：操作流程简化，关键操作（如档案录入、活动发起）步骤不超过3步；表单提交前进行前端校验（如必填项校验、格式校验），校验失败给出明确提示，与后端校验形成双重保障。

- 本地联动：前端直接调用同一工程内的后端接口，无需调用外网接口、无需跨域配置；数据缓存仅用于本地会话，关闭软件后缓存清空，不涉及本地持久化缓存（除数据库存储外）。

- 兼容性约束：适配主流浏览器（Chrome 90\+、Edge 90\+），禁止使用小众浏览器特性，确保村级工作人员常用浏览器可正常访问；前端资源打包后，由Spring Boot服务统一托管，无需单独部署Nginx等前端服务器。

# 四、数据组件约束

## 4\.1 数据库约束

- 数据库类型：固定使用SQLite 3，无需安装独立数据库服务，数据库文件存储于软件本地安装目录，便于单机部署和迁移。

- 表结构约束：严格按照PRD中核心数据字段规范设计表结构，主键统一使用自增ID，唯一字段（如身份证号、家庭户编号）设置唯一约束，避免重复数据；字段类型匹配（如身份证号用VARCHAR，年龄用INT），不冗余设计字段。

- SQL约束：禁止使用复杂SQL语句（如多表联查嵌套、存储过程），简化SQL操作，提升查询效率；所有SQL语句需进行参数化处理，避免SQL注入风险。

- 数据存储：所有业务数据仅存储于本地SQLite数据库，无云端存储、无数据同步功能；不进行任何数据加密处理，数据直接以明文形式存储（贴合需求）；不设计数据备份相关功能，不引入备份组件。

## 4\.2 其他数据组件约束

- 文件存储：村民照片、证明材料等附件，存储于软件本地安装目录下的指定文件夹，路径统一配置，便于管理和访问；附件命名规范（如“身份证\_110101XXXXXX\.jpg”），避免文件名重复。

- 数据校验：后端需对所有输入数据进行校验（如身份证号格式、日期格式、必填项），校验失败返回对应错误提示，与前端校验协同，确保数据准确性。

- 数据异常处理：数据库连接异常、数据查询/插入/更新失败时，后端需捕获异常并输出明确提示，便于开发排查和用户操作，不出现系统崩溃、无响应情况。

# 五、工程初始化目录要求（前后端一体化）

## 5\.2 工程初始化补充要求

- 工程配置：application\.yml中需明确配置SQLite数据库路径（本地路径）、服务端口（建议8080）、JDK版本、前端资源路径等核心配置；禁止配置加密、备份相关参数；配置前端资源托管，确保启动后端服务后可直接访问前端页面。

- 前端打包：开发完成后，将前端代码打包，生成的静态资源放入resources/static目录下，由Spring Boot服务统一托管，启动后端服务即可访问整个软件（前后端一体化运行）。

- 工程说明：README\.md需详细说明开发环境搭建步骤、前后端开发规范、工程启动方法、核心配置说明，便于团队成员快速上手，明确前后端一体化开发、打包、部署流程。
