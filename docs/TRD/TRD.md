# 1. 需求概述

## 1.1 业务背景

随着个人电脑使用场景的不断丰富，用户对本地文件的管理需求日益增长。传统的文件管理器虽能满足基础需求，但在文件预览、在线编辑、Markdown写作等方面功能有限。用户需要一个集成化的本地文件管理解决方案，能够在一款应用中完成文件管理、预览、在线Markdown编写等核心操作。

## 1.2 项目目标

桌面端本地文件管理工具，为个人用户提供文件目录管理、文件操作、文件预览、在线Markdown编写的一体化解决方案。

## 1.3 核心价值

- 一体化文件管理：目录、文件、预览、MD编辑全功能集成
- 离线可用：单机模式，无需网络即可使用
- 便捷部署：双击exe即可运行，无需复杂配置
- 数据安全：本地加密存储、操作日志追踪

## 1.4 适用用户/场景

- 个人用户：需要在本地管理各类文档、资料的普通用户
- 作家/编辑：需要Markdown写作环境的用户
- 学生/教师：需要整理学习资料的用户
- 适用系统：Windows 10/11 操作系统

## 3.2 功能模块设计

### 3.2.1 目录管理模块

#### 功能说明

- **功能点列表**：创建目录、编辑目录（重命名、移动）、删除目录（移入回收站）、查询目录（搜索、面包屑导航、排序）
- **状态机**：
  ```
  普通目录 → 删除 → 回收站 → 恢复 → 普通目录
            ↘ 永久删除（30天后自动/手动清空）
  ```
- **业务规则**：
  - 目录名称1-50字符，禁止特殊字符：`/ \ : * ? " < > |`
  - 同一父目录下目录名唯一
  - 删除目录时，其下所有子目录和文件一并移入回收站
  - 回收站保留30天后自动永久删除

#### 界面设计

- **页面结构**：目录树侧边栏 + 主内容区列表 + 顶部工具栏
- **关键组件**：面包屑导航、目录列表、新建目录按钮、右键菜单（重命名/移动/删除）
- **交互流程**：点击目录 → 展示子目录/文件列表 → 右键操作
- **异常提示**：重名提示"当前目录下已存在同名目录"、非法字符提示禁止字符列表

#### 接口设计

| 接口 | 方法 | 功能 |
|------|------|------|
| `/api/directory/create` | POST | 创建目录 |
| `/api/directory/update` | PUT | 更新目录 |
| `/api/directory/delete` | DELETE | 删除目录 |
| `/api/directory/list` | GET | 目录列表 |
| `/api/directory/tree` | GET | 目录树 |
| `/api/directory/search` | GET | 搜索目录 |

**创建目录请求示例**
```json
{
  "name": "新建目录",
  "parentId": 1
}
```

**创建目录响应示例**
```json
{
  "code": 0,
  "data": {
    "id": 1,
    "name": "新建目录",
    "parentId": 1,
    "createdAt": "2026-04-16 10:00:00"
  }
}
```

#### 数据设计

**目录表 (t_directory)**

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 目录ID |
| name | VARCHAR(50) | NOT NULL | 目录名称 |
| parent_id | BIGINT | FK→t_directory(id) | 父目录ID |
| user_id | BIGINT | FK→t_user(id) | 用户ID |
| path | VARCHAR(500) | - | 完整路径 |
| created_at | DATETIME | NOT NULL | 创建时间 |
| updated_at | DATETIME | NOT NULL | 更新时间 |
| deleted | TINYINT | DEFAULT 0 | 逻辑删除 |
| delete_at | DATETIME | - | 删除时间(回收站) |

**索引**
```sql
INDEX idx_parent_id (parent_id)
INDEX idx_user_id (user_id)
```

#### 性能设计

- 目录列表加载 ≤ 1秒（1000项内）
- 目录树加载 ≤ 1秒
- 实现：数据库索引 + 缓存目录结构

#### 安全设计

- 删除操作需二次确认弹窗
- 目录名称特殊字符过滤（白名单过滤）
- 路径穿越防护：`../` 过滤

#### 测试方案

- **功能测试**：创建/重命名/移动/删除/恢复目录
- **边界测试**：目录名50字符、特殊字符、嵌套深度
- **接口测试**：正常请求、异常请求

#### 验收标准

- **验收条件**：
  1. 创建目录成功 → 目录出现在列表中
  2. 重名创建 → 提示"重名"错误
  3. 使用特殊字符 → 提示"禁止使用"错误
  4. 删除目录 → 弹窗确认 → 目录移入回收站
  5. 恢复目录 → 目录恢复正常可见
- **验证方式**：UI操作 + 接口返回验证
- **成功判定**：所有功能点测试通过

### 3.2.2 文件管理模块

#### 功能说明

- **功能点列表**：上传文件、编辑文件（重命名、移动、替换内容）、删除文件（移入回收站）、查询文件、下载文件（单个/批量ZIP）
- **状态机**：
  ```
  普通文件 → 删除 → 回收站 → 恢复 → 普通文件
             ↘ 永久删除（30天后自动/手动清空）
  ```
- **业务规则**：
  - 单个文件≤100MB，批量≤50个/次
  - 支持格式：doc、docx、xls、xlsx、ppt、pptx、pdf、txt、md、jpg、jpeg、png、gif、bmp、mp3、wav、mp4、avi
  - MD5相同则拒绝上传（同一目录下）
  - 删除文件移入回收站，保留30天
  - 回收站支持一键清空（不可恢复）

#### 界面设计

- **页面结构**：工具栏 + 文件列表 + 底部状态栏
- **关键组件**：上传按钮、文件列表表格、拖拽区、右键菜单、下载按钮
- **交互流程**：选择文件 → 上传 → 进度显示 → 完成
- **异常提示**："文件大小超过限制"、"格式不支持"、"上传失败"

#### 接口设计

| 接口 | 方法 | 功能 |
|------|------|------|
| `/api/file/upload` | POST | 上传文件 |
| `/api/file/update` | PUT | 更新文件信息 |
| `/api/file/delete` | DELETE | 删除文件 |
| `/api/file/list` | GET | 文件列表 |
| `/api/file/download` | GET | 下载文件 |
| `/api/file/batch-download` | GET | 批量下载ZIP |
| `/api/file/search` | GET | 搜索文件 |

**上传文件请求**
- `file`: File (multipart/form-data)
- `parentId`: Long (可选，默认当前目录)

**文件列表响应**
```json
{
  "code": 0,
  "data": {
    "list": [
      {
        "id": 1,
        "name": "test.pdf",
        "size": 1024000,
        "format": "pdf",
        "createdAt": "2026-04-16 10:00:00"
      }
    ],
    "total": 100,
    "page": 1
  }
}
```

#### 数据设计

**文件表 (t_file)**

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 文件ID |
| name | VARCHAR(50) | NOT NULL | 文件名 |
| ext | VARCHAR(20) | - | 扩展名 |
| size | BIGINT | NOT NULL | 文件大小 |
| md5 | VARCHAR(32) | NOT NULL | MD5校验值 |
| storage_path | VARCHAR(500) | NOT NULL | 存储路径 |
| parent_id | BIGINT | FK→t_directory(id) | 所属目录 |
| user_id | BIGINT | FK→t_user(id) | 用户ID |
| created_at | DATETIME | NOT NULL | 上传时间 |
| updated_at | DATETIME | NOT NULL | 更新时间 |
| deleted | TINYINT | DEFAULT 0 | 逻辑删除 |
| delete_at | DATETIME | - | 删除时间(回收站) |

**索引**
```sql
INDEX idx_parent_id (parent_id)
INDEX idx_md5_parent (md5, parent_id)  -- MD5去重
```

#### 性能设计

- 10MB文件上传 ≤ 10秒
- 列表加载（1000项）≤ 1秒
- 批量下载：流式处理，避免内存溢出
- 实现：分片上传、数据库索引优化

#### 安全设计

- 文件名特殊字符过滤
- 路径穿越防护：`../` 过滤
- 上传格式白名单验证
- 删除操作二次确认
- 文件加密存储（AES-256）【待产品确认是否实现】

#### 测试方案

- **功能测试**：上传、下载、删除、恢复、搜索
- **边界测试**：100MB文件、50个批量、特殊字符、无效格式
- **接口测试**：正常请求、超大文件、并发上传

#### 验收标准

- **验收条件**：
  1. 上传≤100MB成功 → 进度显示 → 成功提示
  2. MD5相同拒绝 → 提示"文件已存在"
  3. 删除文件 → 弹窗确认 → 移入回收站
  4. 批量下载 → 生成ZIP → 自动下载
- **验证方式**：UI操作 + 接口返回 + 文件系统验证
- **成功判定**：所有功能点测试通过

#### 界面设计

- **页面结构**：工具栏 + 编辑区 + 预览区
- **关键组件**：Markdown编辑器(EasyMDE)、工具栏按钮、预览区
- **交互流程**：创建MD → 进入编辑 → 实时预览 → 保存/导出

  2. 编辑内容 → 右侧预览实时显示
  3. 30秒后 → "已自动保存"提示
  4. 导出PDF → 格式保持、排版正确
- **验证方式**：编辑操作 + 导出文件验证
- **成功判定**：导出文件与预览一致

### 3.2.5 回收站模块

#### 功能说明

- **功能点列表**：查看回收站内容、恢复文件/目录、清空回收站、自动清理
- **状态机**：
  ```
  回收站 → 恢复 → 恢复到原目录
        ↘ 30天后自动删除
        ↘ 手动清空 → 不可恢复
  ```
- **业务规则**：
  - 保留30天后自动永久删除
  - 恢复时回到原目录位置
  - 清空操作不可恢复，需二次确认

#### 界面设计

- **页面结构**：回收站列表 + 工具栏
- **关键组件**：回收站列表、全选框、恢复按钮、清空按钮
- **交互流程**：进入回收站 → 选择恢复/清空 → 执行
- **异常提示**：清空确认提示、成功/失败提示

#### 接口设计

| 接口 | 方法 | 功能 |
|------|------|------|
| `/api/trash/list` | GET | 回收站列表 |
| `/api/trash/restore/{id}` | POST | 恢复文件 |
| `/api/trash/clean` | DELETE | 清空回收站 |
| `/api/trash/auto-clean` | POST | 触发自动清理 |

**回收站列表响应**
```json
{
  "code": 0,
  "data": [
    {
      "id": 1,
      "name": "test.pdf",
      "originalPath": "/文档/test.pdf",
      "type": "file",
      "deleteAt": "2026-04-16 10:00:00",
      "expireAt": "2026-05-16 10:00:00"
    }
  ]
}
```

#### 数据设计

使用 `t_directory` 和 `t_file` 表的 `deleted`、`delete_at` 字段，无需新表。

**定时清理查询**
```sql
SELECT * FROM t_directory WHERE deleted=1 AND delete_at < NOW() - 30 days
SELECT * FROM t_file WHERE deleted=1 AND delete_at < NOW() - 30 days
```

#### 性能设计

- 回收站查询 ≤ 1秒
- 实现：索引优化、定时任务低峰期执行

#### 安全设计

- 清空操作二次确认
- 恢复需验证原目录是否存在
- 操作日志记录

#### 测试方案

- **功能测试**：恢复、清空、查看
- **边界测试**：30天临界、批量恢复、原目录不存在
- **接口测试**：正常操作、异常恢复

#### 验收标准

- **验收条件**：
  1. 删除文件 → 回收站显示
  2. 恢复文件 → 回到原目录
  3. 手动清空 → 确认后清空
  4. 30天后 → 自动删除
- **验证方式**：回收站操作 + 数据库验证
- **成功判定**：功能正常运作
