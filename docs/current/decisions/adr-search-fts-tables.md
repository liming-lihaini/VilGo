# ADR-6-3: 全文检索 FTS5 表设计

## 背景

使用 SQLite FTS5 实现全文检索，需要设计索引表结构。

## 方案

### 方案 A: 单一 FTS 表

文件 + 目录共用一个 FTS 表，通过 type 字段区分。

- 优点: 查询简单，一次搜索可返回文件和目录
- 缺点: 类型筛选需额外处理

### 方案 B: 分离 FTS 表（推荐）

文件和目录分别使用独立的 FTS 表。

- 优点: 类型筛选高效、结构清晰
- 缺点: 需要分别维护两个表

## 选择

选择: 方案 B（分离 FTS 表）

原因: 目录搜索和文件搜索有不同需求（目录无需内容搜索），分离设计更清晰

## 影响

- 创建两个 FTS 虚拟表: file_fts, directory_fts
- 需要分别创建同步触发器
