# ADR-6-2: 搜索历史存储方式

## 背景

Issue #6 需要实现搜索历史功能，记录最近10条搜索关键词。需要确定存储方式。

## 方案

### 方案 A: 数据库存储

使用 SQLite 数据库表存储搜索历史。

- 优点: 数据持久化、跨会话保留
- 缺点: 增加数据库复杂度

### 方案 B: 本地存储

使用浏览器 localStorage 存储搜索历史。

- 优点: 实现简单、不增加后端复杂度
- 缺点: 仅限单浏览器会话

## 选择

选择: 方案 B（本地存储）

原因: MVP 阶段简单优先，搜索历史非核心数据，单机使用场景下 localStorage 足够

## 影响

- 前端维护搜索历史
- 后端无需提供历史存储 API（可选提供）
