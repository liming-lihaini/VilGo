# ADR-6-1: 全文检索索引更新策略

## 背景

Issue #6 需要实现全文检索功能，使用 SQLite FTS5。核心问题是：当文件创建/修改/删除时，如何同步更新 FTS5 索引？

## 方案

### 方案 A: 实时更新

每次文件操作（创建/修改/删除）时，立即更新 FTS5 索引。

- 优点: 搜索结果实时准确
- 缺点: 文件操作性能受影响

### 方案 B: 定时更新（推荐）

使用定时任务（如每5分钟）批量同步 FTS5 索引。

- 优点: 不影响文件操作性能，实现简单
- 缺点: 搜索结果存在短暂延迟

## 选择

选择: 方案 B（定时更新）

原因: MVP 阶段优先保证文件操作性能，允许短暂搜索延迟

## 影响

- 需要创建定时同步任务
- 搜索接口需要处理索引与实际数据的差异
