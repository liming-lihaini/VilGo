# 计划阶段知识

## 1. 阶段目标

将设计决策转化为**可执行计划**，明确 Task 列表、依赖关系、验收条件。

---

## 2. 任务拆分原则

### 2.1 Task 定义

| 字段 | 说明 |
|------|------|
| Task ID | 唯一标识，顺序编号 |
| 内容 | 具体要实现的内容 |
| 依赖 | 依赖的其他 Task |
| 验收条件 | 如何验证完成 |

### 2.2 拆分粒度

| 粒度 | 说明 |
|------|------|
| 推荐 | 每个 Task 涉及 1-2 个文件 |
| 最小 | 可独立实现和验证 |
| 最大 | 不超过 1 天工作量 |

### 2.3 拆分维度

| 维度 | 说明 |
|------|------|
| 模块拆分 | 按功能模块拆分 |
| 层级拆分 | 按前后端拆分 |
| 依赖拆分 | 按依赖关系排序 |

---

## 3. 实现计划模板

```markdown
# {日期} {功能} 实现计划

## 概述
{简要描述实现的功能}

## Task 列表

### Task 1: {任务名称}
- **内容**: {具体实现内容}
- **文件**: {涉及的文件路径}
- **依赖**: {依赖的 Task}
- **验收条件**:
  1. {验收点1}
  2. {验收点2}

### Task 2: {任务名称}
...

## 依赖关系图
{描述 Task 之间的依赖}

## 验收条件映射
| 原始条件 | 对应 Task |
|----------|-----------|
| 条件1 | Task 1, Task 2 |
| 条件2 | Task 3 |
```

## 4. 验收条件

### 4.1 原则

| 原则 | 说明 |
|------|------|
| 明确 | 可客观判定通过/失败 |
| 可测 | 可执行验证 |
| 具体 | 涉及文件、方法、字段 |

### 4.2 验收条件模板

```markdown
### 验收条件
- [ ] 条件1: {验证方法}
- [ ] 条件2: {验证方法}
```

### 4.3 示例

```markdown
### 验收条件
- [ ] SearchController 提供 /api/search 接口
- [ ] SearchService 实现 searchAll() 方法
- [ ] 前端 SearchView.vue 渲染搜索结果
```

## 5. 本阶段执行标准

### 5.1 输入要求

| 输入项 | 来源 | 说明 |
|--------|------|------|
| Task | pipeline-state | 需求理解结果 |
| Boundaries | pipeline-state | 范围约束 |
| Decisions | pipeline-state | ADR 决策文档 |
| ADR | docs/current/decisions/ | 技术决策详情 |
| 阶段知识 | docs/knowledge/stages/plan.md | 本阶段执行指引 |

### 5.2 执行步骤

1. **分析 ADR** - 理解设计决策
2. **拆分 Task** - 按维度拆分为可执行任务
3. **确定依赖** - 明确任务间依赖关系
4. **映射验收** - 将原始验收条件映射到 Task
5. **编写计划** - 按模板输出实现计划
6. **更新 State** - 填写 Task 列表

### 5.3 输出要求

| 输出项 | 内容 |
|--------|------|
| 实现计划 | docs/plans/{date}-{功能}实现计划.md |
| Task 列表 | pipeline-state 中的 Current Stage |

### 5.4 验收条件

| 条件 | 判定标准 |
|------|----------|
| 任务完整 | 所有 ADR 决策都有对应 Task |
| 依赖清晰 | Task 依赖关系明确，无循环依赖 |
| 验收可测 | 每个 Task 有明确的验收条件 |
| 映射完整 | 原始验收条件全部覆盖 |

## 6. 执行检查清单

### 6.1 任务拆分检查

- [ ] Task 粒度合适（1-2个文件）
- [ ] 依赖关系清晰
- [ ] 无遗漏的功能点

### 6.2 验收条件检查

- [ ] 每个 Task 有验收条件
- [ ] 验收条件可测试
- [ ] 原始条件全部覆盖

### 6.3 计划完整性检查

- [ ] 包含数据库变更
- [ ] 包含后端代码
- [ ] 包含前端代码
- [ ] 包含配置变更
