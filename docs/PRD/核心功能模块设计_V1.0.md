## 2. 项目背景

面向村级工作人员（两委成员、网格员）及村民的单机版数字化管理工具，聚焦村级日常管理痛点（档案混乱、信息零散、统计繁琐、公示不便等），实现村级管理全流程数字化、操作简易化、服务便民化。界面设计贴合中老年用户使用习惯，无需专业计算机基础即可上手。

当前存在的主要痛点：
- 档案管理混乱：村民信息、户籍信息缺乏统一规范化管理
- 信息零散：各类数据分散，缺乏关联和整合
- 统计繁琐：人工统计工作量大、效率低
- 公示不便：信息发布和反馈渠道不畅

## 3. 业务目标

- **功能目标**：覆盖9大核心模块，涵盖人事、户籍、党务、活动等村级管理全场景，实现核心业务流程数字化，满足日常办公需求
- **体验目标**：界面简洁、操作便捷，减少冗余步骤；单机部署无需复杂配置，安装即能用
- **技术目标**：基于Java+SQLite架构，本地独立运行，无外网依赖，运行稳定、响应流畅
- **业务目标**：规范村级管理流程，减少人工录入、统计工作量；实现数据可查可追溯，为村级决策提供数据支撑

## 5. 业务流程

### 5.1 主流程

**村民档案管理流程**：
1. 工作人员新增村民档案（录入基础信息、保障信息）
2. 系统校验身份证号唯一性
3. 档案入库并生成档案编号
4. 支持后续查询、编辑、统计、导出

**户籍管理流程**：
1. 创建家庭户（自动关联户主档案）
2. 添加/删除/修改家庭成员
3. 记录年度收入台账
4. 记录户籍变动并生成留痕

**公益活动流程**：
1. 工作人员发起活动（填写名称、时间、地点、内容）
2. 村民报名（工作人员协助录入）
3. 活动出勤签到登记
4. 活动归档并统计

### 5.2 分支流程

- 销户流程：档案标记销户状态，保留历史数据
- 分户/合户流程：原家庭户变更，新家庭户创建
- 帮扶流程：特殊人群 → 分配帮扶责任人 → 记录帮扶措施 → 成效评估

### 5.3 异常流程

| 异常类型 | 触发场景 | 处理方式 |
|----------|----------|----------|
| 重复录入 | 身份证号/家庭户编号重复 | 提示"已存在相关记录，请核对后重新录入" |
| 信息缺失 | 必填字段未填写 | 提示"请补充必填信息（标注*字段）" |
| 数据格式错误 | 日期/时长/金额格式错误 | 提示"输入格式无效，请按规范填写" |
| 查询无结果 | 条件查询无匹配数据 | 显示"暂无相关记录，请调整查询条件后重试" |
| 操作异常 | 删除已关联数据 | 提示"该数据已关联其他记录，无法直接操作" |

## 6. 功能详细说明

### 6.1 村民人事档案管理

**输入**：
- 基础信息：姓名、身份证号（唯一）、性别、出生日期、联系方式、家庭住址、婚姻状况、政治面貌、照片、备注
- 保障信息：保障类型（无/社保/养老/低保/其他）
- 人员信息：是否户主、人员类型（群众/党员/预备党员/在读学生/军人/其他）、人员角色（管理员/普通用户）
- 录入人、录入时间

**处理逻辑**：
1. 新增/编辑档案信息
2. 身份证号唯一性校验
3. 关联户籍档案（为户主时创建家庭户）
4. 持久化存储

**输出**：
- 档案详情、查询结果列表、统计报表、Excel导出文件

**约束条件**：
- 身份证号为全局唯一标识，不可重复
- 销户后保留历史记录，软删除标记

### 6.2 户籍档案管理

**输入**：
- 家庭户编号（唯一）、户主ID（关联人事档案）、家庭住址、附件材料（图片/PDF/json）
- 家庭成员列表（关联人事档案ID、关系）
- 年度收入（年份、总收入、人均收入、收入来源）
- 户籍变动（变动类型、时间、原因）

**处理逻辑**：
1. 同步获取户主信息（无需重复录入）
2. 管理家庭成员增删改查
3. 记录年度收入台账
4. 生成户籍变动留痕

**输出**：
- 家庭户详情、人口统计报表

**约束条件**：
- 家庭户编号全局唯一
- 成员变动保留完整历史轨迹

### 6.3 特殊人群档案管理

**输入**：
- 身份证号（关联人事档案）、人群类型（脱贫户/监测户/残疾人/孤寡老人/其他）
- 帮扶责任人ID（关联两委成员）
- 帮扶措施、帮扶时间、帮扶成效
- 帮扶状态（进行中/已完成）

**处理逻辑**：
1. 关联人事档案获取基础信息
2. 分配帮扶责任人
3. 记录帮扶过程
4. 任务到期提醒

**输出**：
- 特殊人群档案、帮扶台账、帮扶报表、任务提醒

### 6.4 两委班子管理

**输入**：
- 成员姓名、性别、职务、联系方式、分管工作、入职时间
- 任务分配（任务内容、分配时间、完成期限、完成状态、完成成效）
- 会议记录（会议时间、地点、参会人员、决议事项、落实情况）

**输出**：
- 成员信息列表、任务进度追踪、会议纪要

### 6.5 公益活动管理

**输入**：
- 活动名称、活动时间、活动地点、活动内容、报名要求
- 参与人员、出勤时长、工值
- 活动状态（未开始/进行中/已归档）、活动总结

**处理逻辑**：
1. 创建并发布活动
2. 记录活动参与人
3. 签到出勤记录
4. 活动归档统计

**输出**：
- 活动列表、活动状态、参与统计、出勤统计

### 6.6 党务管理

**输入**：
- 身份证号（关联人事档案）、入党时间���转��时间、党员状态（正式/预备/积极分子）
- 党务活动（活动主题、参与情况）

**输出**：
- 党员档案、党务活动记录

### 6.7 村委公示栏

**输入**：
- 公示标题、公示内容（支持图文）
- 公示类型（通知/政策文件/财务公开/项目公示/惠民信息）
- 发布人、公示期限

**处理逻辑**：
1. 发布公示内容
2. 村民查看
3. 反馈收集与回复

**输出**：
- 公示列表、反馈记录

### 6.8 新闻动态

**输入**：
- 标题、内容（支持图文）
- 分类标签（政策解读/活动报道等）
- 发布人、发布时间

**输出**：
- 新闻列表（按分类/时间排序）

### 6.9 数字大屏

**数据来源**：各模块实时数据聚合

**展示内容**：
- 总人口数、家庭户数
- 人员结构（按类型分布）
- 特殊人群数量及帮扶进度
- 两委任务完成情况
- 公益活动统计（近期活动、参与人数）
- 公示及新闻更新提醒
- 户籍变动统计

## 10. 异常流程与边界情况

| 异常类型 | 场景描述 | 处理方式 |
|----------|----------|----------|
| 重复录入 | 身份证号/家庭户编号重复 | 提示"已存在相关记录，请核对后重新录入"，阻止提交 |
| 信息缺失 | 必填字段未填写 | 提示"请补充必填信息（标注*字段）"，高亮必填项 |
| 数据格式错误 | 日期/时长/金额格式错误 | 提示"输入格式无效，请按规范填写"，显示正确格式 |
| 查询无结果 | 搜索条件无匹配 | 显示"暂无相关记录，请调整查询条件后重试" |
| 操作异常 | 删除已关联数据 | 提示"该数据已关联其他记录，无法直接操作，请先解除关联" |
